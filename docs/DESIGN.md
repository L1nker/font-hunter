# Capítulo 1 — Filosofia do Design

---

# Objetivo

O objetivo deste documento é definir a identidade visual do Font Hunter.

Ele descreve como a interface deve parecer, se comportar e evoluir ao longo do desenvolvimento.

Nenhum componente poderá ser criado sem respeitar estas diretrizes.

---

# Personalidade do Produto

O Font Hunter deve transmitir cinco sensações principais.

## Simplicidade

O usuário nunca deve se sentir perdido.

Cada tela deve possuir um objetivo claro.

Poucos elementos.

Muito espaço em branco.

Nenhuma informação desnecessária.

---

## Profissionalismo

A ferramenta deve transmitir confiança.

O usuário deve sentir que está utilizando um software desenvolvido com cuidado.

Evitar aparência de projeto acadêmico ou experimental.

---

## Velocidade

Mesmo quando uma análise demorar alguns segundos, a interface deve transmitir sensação de rapidez.

O usuário deve perceber progresso constante.

Jamais deixar a interface "congelada".

---

## Organização

Todas as informações devem possuir hierarquia visual.

O usuário deve identificar rapidamente:

* resultado principal;

* alternativas;

* ações disponíveis;

* informações secundárias.

---

## Elegância

A interface deve ser discreta.

O design nunca deve competir com o conteúdo.

A tipografia e os resultados são os protagonistas.

---

# Inspirações

O Font Hunter utiliza como referência visual:

* Google Fonts
* Figma
* GitHub
* Notion

Essas plataformas compartilham características importantes:

* minimalismo;

* clareza;

* excelente legibilidade;

* foco no conteúdo;

* baixa poluição visual.

---

# O que NÃO queremos

O Font Hunter não deve parecer:

* um painel administrativo;

* um marketplace;

* uma loja virtual;

* uma rede social;

* um site repleto de anúncios;

* um software corporativo antigo.

---

# Filosofia da Interface

Toda decisão visual deverá responder:

> Isso ajuda o usuário a encontrar uma fonte?

Se a resposta for negativa, o elemento deverá ser removido ou simplificado.

---

# Princípio da Invisibilidade

O melhor design é aquele que desaparece.

O usuário deve prestar atenção:

* na imagem;

* nas fontes;

* na comparação.

Nunca na interface.

---

# Hierarquia Visual

A ordem de importância da interface será sempre:

1. Imagem enviada

2. Resultado identificado

3. Alternativas

4. Preview

5. Filtros

6. Informações técnicas

Toda decisão visual deverá respeitar essa hierarquia.

---

# Uso de Espaços

Espaços em branco fazem parte da interface.

Eles não representam áreas vazias.

Representam organização.

Jamais preencher espaços apenas porque "parecem vazios".

---

# Densidade da Interface

A interface deverá manter baixa densidade de informações.

Sempre que possível utilizar:

* menus expansíveis;

* acordeões;

* popovers;

* tooltips;

para esconder detalhes técnicos.

---

# Consistência

Todos os componentes semelhantes deverão possuir exatamente o mesmo comportamento.

Exemplo:

Todos os botões primários utilizam a mesma cor.

Todos os cards utilizam a mesma sombra.

Todos os campos possuem o mesmo arredondamento.

---

# Clareza

O usuário nunca deve interpretar.

Ele deve compreender imediatamente.

Ícones importantes deverão possuir texto.

Evitar símbolos pouco conhecidos.

---

# Sensação Final

Ao utilizar o Font Hunter pela primeira vez, o usuário deve pensar:

> "É simples."

Após alguns minutos:

> "É rápido."

Ao concluir a pesquisa:

> "Encontrei tudo o que precisava."

Essa é a experiência que deverá orientar todas as decisões visuais futuras.

# Capítulo 3 — Design Tokens

---

# Objetivo

Este capítulo define todos os tokens visuais oficiais do Font Hunter.

Os tokens representam a identidade visual do sistema.

Nenhum componente deverá utilizar valores fixos quando existir um token correspondente.

Todo o projeto deverá referenciar estes tokens, garantindo consistência e facilitando futuras alterações.

---

# O que são Design Tokens

Design Tokens são nomes padronizados que representam propriedades visuais do sistema.

Em vez de escrever diretamente:

```css
color: #2563EB;
```

O projeto deverá utilizar:

```css
color: var(--color-primary);
```

Isso permite alterar toda a identidade visual modificando apenas um único local.

---

# Princípios

Os Design Tokens devem ser:

* reutilizáveis;
* independentes de tecnologia;
* consistentes;
* fáceis de compreender;
* semanticamente nomeados.

Jamais utilizar nomes como:

* azul1
* azul2
* cinzaEscuro3

Sempre utilizar nomes que representem função.

Todos os componentes deverão seguir este documento.

Componentes devem ser:
